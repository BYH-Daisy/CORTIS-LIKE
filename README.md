## 一、项目创意构思

### 设计理念

本项目的核心目标是为一支名为 **CORTIS** 的韩国男子偶像团体打造一面"安利墙"。所谓"安利"，就是让一个原本不认识这个团体的人，通过浏览页面逐步产生兴趣。由此衍生出明确的浏览动线：

```
介绍团队 → 展示作品 → 欣赏影像 → 表达喜爱
```

对应到页面结构：

| 动线环节     | 页面模块                 | 设计意图                            |
| -------- | -------------------- | ------------------------------- |
| **介绍**   | Cover 区 + BubbleMenu | 封面视觉冲击 + 成员目录导航，让访客第一眼认识 CORTIS |
| **展示作品** | Music 区              | 3D 轮播展示 5 首原创歌曲，配套歌词面板沉浸试听      |
| **欣赏影像** | Danmaku 区幻灯片         | 48 张照片轮播，记录团队点滴                 |
| **表达喜爱** | 弹幕 + 点赞              | 访客可发送弹幕评论、点赞照片，互动反馈             |

### 视觉风格

- **主色调**：绿色系（`#2e7d32` / `#4caf50`），呼应乐队青春活力的气质
- **花体字标题**：Dancing Script 字体，营造手写感与亲和力
- **玻璃态卡片**：音乐轮播和歌词面板采用毛玻璃效果（`backdrop-filter: blur`）
- **动画体系**：BlurText 逐字模糊入场、GSAP 泡泡弹出、ClickSpark 点击粒子，节奏有缓有急

## 二、技术方案简述

### 技术栈

| 技术                          | 用途        |
| --------------------------- | --------- |
| React 19 + Vite 8           | 前端框架与构建工具 |
| React Router 7 (HashRouter) | 前端路由      |
| motion (framer-motion)      | 3D 轮播拖拽动画 |
| GSAP 3                      | 泡泡菜单入场动画  |
| react-icons                 | 轮播图标      |

### 组件架构

```
src/
├── main.jsx           # 入口，挂载 AudioProvider + HashRouter 路由
├── App.jsx            # 主页面（Cover → Sprout → Music → Danmaku）
├── App.css            # 主页面样式
├── index.css          # 全局重置样式
├── AudioContext.jsx   # 音频状态管理 Context（跨路由持续播放）
├── BlurText.jsx       # 逐字模糊入场文字动画
├── BubbleMenu.jsx     # GSAP 驱动的泡泡菜单导航
├── BubbleMenu.css     # 泡泡菜单样式
├── Carousel.jsx       # motion 驱动的 3D 歌曲轮播
├── Carousel.css       # 轮播样式
├── ClickSpark.jsx     # Canvas 点击粒子特效
├── MemberPage.jsx     # 成员个人介绍页（5 个路由）
└── MusicPlayer.jsx    # 独立播放器（备用，未启用）
```

### 数据流

- 弹幕数据通过 `useState` 管理，初始预置 7 条默认弹幕，用户可追加，最多保留 20 条
- 点赞数据分两组：幻灯片点赞 (`likeCounts`) 和歌曲点赞 (`songLikes`)，支持长按连发 + 爱心飘出动画
- 音频播放通过 `AudioContext` 提升到路由外层，`audioRef` + 时间更新事件驱动歌词高亮和进度条，支持跨路由持续播放
- 歌词解析 `[mm:ss:ff][mm:ss:ff]` 格式时间戳，韩文 + 中文成对显示，唱到行放大加粗白色，过往行降低透明度
- 轮播用 `useMotionValue` + `useTransform` 实现 3D 旋转拖拽，支持无限循环，滑动时静默切换音频源（不自动播放）

## 三、AI 协作过程概述

整个开发过程采用"需求描述 → AI 生成代码 → 人工验证 → 反馈修正"的迭代模式，共经历 14 轮关键交互：

| 轮次  | 需求               | AI 产出                                     | 迭代要点                            |
| --- | ---------------- | ----------------------------------------- | ------------------------------- |
| 1   | 弹幕组件             | 弹幕层 + 输入发送                                | 全屏定位修复                          |
| 2   | 点赞按钮             | 点赞 + 爱心飘出动画                               | 长按连发                            |
| 3   | 气泡目录动画           | BubbleMenu + GSAP 动画                      | 路由打通                            |
| 4   | 音乐播放器            | Carousel 3D 轮播 + 音频联动                     | 歌词跟随修复                          |
| 5   | 细节打磨             | BlurText / ClickSpark / 响应式               | 多轮微调                            |
| 6   | 轮播暂停修复           | onClick 从 stage 移到 img                    | 点击弹幕不干扰轮播                       |
| 7   | 内置弹幕             | 7 条默认弹幕 + 字号/标题样式调整                       | 开屏即显                            |
| 8   | 项目重命名            | 全量改名 + 最终部署验证                             | 交付                              |
| 9   | GO! 音频替换 + 时间戳歌词 | 按 `[mm:ss:ff]` 解析时间戳，唱到行高亮                | 时间戳格式纠错                         |
| 10  | 歌词时间戳纠错 + 跳动修复   | 修正 3 处 end<start，滚动改为 smooth              | 切歌清空旧数据                         |
| 11  | 歌曲 2~5 音频替换      | 替换 FaSHioN/JoyRide/Lullaby 音频+歌词          | 修复多处 end<start、缺冒号、前导空格         |
| 12  | 跨路由音频持续播放        | 创建 AudioContext 提升到 Router 外层             | Carousel 初始位置保护 + 移除 StrictMode |
| 13  | 活跃歌词中文白色         | CSS 添加 `.lyric-block.active .lyric-cn` 白色 | —                               |
| 14  | 页面水印             | JS 生成散落水印，6×3 网格随机偏移                      | Cover/成员页无水印                    |

AI 在过程中承担了代码生成、Bug 定位、方案建议等角色；人工负责需求定义、设计决策和验收测试。

## 四、Skill 技能包使用说明

在开发过程中调用了 opencode 提供的 **frontend-design** 技能包辅助设计决策。

### 选用的 Skill

**frontend-design** — 前端设计引导技能

### 使用方式

1. **设计定位**：技能指导从主题出发做设计决策。根据"安利墙"这一主题，确定了"介绍 → 作品 → 影像 → 互动"的浏览动线，并围绕 CORTIS 的气质选用绿色系 + Dancing Script 花体字，避免模板化配色方案。

2. **结构即信息**：技能强调"结构编码真实信息"。页面严格按"先认识人、再听歌、最后看照片发弹幕"的认知顺序组织，与安利逻辑一致。

3. **动效有节制**：技能建议"有编排的瞬间比散落的效果更有力"。页面只在关键节点使用动画（封面 BlurText 入场、GSAP 泡泡弹开、点击粒子），不做过量堆砌。

4. **响应式与质量底线**：技能要求"移动端可用的质量底线"。弹幕区和音乐区均做了 `@media (max-width: 520px)` 适配。

通过在 AI 协作中引入设计技能的约束，避免了常见的 AI 生成模板感（暖白背景 + 陶土色点缀的默认组合），使页面有了围绕 CORTIS 这一特定主题的视觉辨识度。

---

### 特色功能

- **跨路由音频播放**：AudioContext 将音频状态提升到 Router 外层，浏览成员页时音乐不中断
- **时间轴歌词联动**：`[mm:ss:ff]` 格式时间戳解析，唱到行放大高亮 + 中文翻译纯白显示，过往行淡化
- **3D 歌曲轮播**：motion 驱动的 3D 旋转拖拽，5 首歌曲循环滑动，支持拖拽和指示器导航
- **弹幕系统**：7 条默认弹幕自动循环发射，用户可发送新弹幕，随机颜色 + 全屏滚动
- **点赞特效**：长按连发 + 随机爱心 emoji 飘出动画
- **泡泡菜单导航**：GSAP 驱动弹性入场动画，点击跳转 5 个成员个人介绍页
- **页面水印**：JS 散乱生成 18 个水印图案，6×3 网格随机偏移，Cover 和成员页自动隐藏
